## 🤖 AIML (AI Markup Language)

### 1. <text> (文本)
功能：输出普通对话。
示例：`<text>任务已完成。</text>`

### 2. <file> (文件)
功能：写本地文件。
属性：`path` (必填); `type` (可选: all/diff,没有read 默认 all)。
示例：
```xml
<file path="src/main.ts" type="all">内容</file>
<file path="pkg.json" type="diff">@@ -1,1 +1,2 @@</file>
```

### 3. <act> (命令)

功能：执行终端命令，要求使用标准的 PowerShell 语法。
属性：`window` (可选: true/false, 默认 false。设为 true 时将在物理新窗口独立运行，不阻塞主进程，适用于 dev/serve 服务)。
示例：`<act>npm install react react-dom ; npm run build</act>`
示例：`<act window="true">npm run dev</act>`


### 5. <continue> (续传)

功能：当输出体量过大时，AI 在当前批次结尾输出此标签，系统解析到后会自动回复触发词，触发 AI 输出下一批次内容。也可以实现分多次输出内容。
属性：`reply` (可选，自定义系统回复给 AI 的提示词，默认："继续")。
示例：`<continue reply="输出6个文件的完整代码，分2次输出，要求最小侵入，非必要不修改。" />`

### 6. <upload> (文件挂载)

功能：将本地的某个文件挂载到下一次对话的上下文中，不支持目录
属性：`path` (路径); `grid` (可选: true/false, 默认 false，仅对图片有效，是否添加辅助坐标网格)。
示例：`<upload path="C:\Users\Admin\Desktop\error_screenshot.jpg" grid="false" />`
