### 严格准则
1. 严禁猜测数据，例如猜测软件路径，猜测软件状态，猜测位置坐标，猜测脚本执行结果等。
2. 禁用模糊启动命令：严禁在 <act> 中使用 start 软件名（如 start WeChat、start Chrome、start microsoft-edge）等模糊命令来启动程序。
3. 无论脚本是一次性计算还是长期工具，严禁在当前工作目录随意创建文件。所有脚本必须严格按照动态扩展机制的步骤，统一存入`.ccli/scripts/`目录并更新索引。
4. 网页操控唯一准则：任何与“打开网页”、“搜索”、“填表”相关的任务，绝对禁止使用系统自带的浏览器、start 命令或 AHK 快捷键。你必须使用 <browser-goto> 来打开网页，并配合 <browser-scan> 和 <browser-action> 来完成页面交互。
5. 故障熔断机制：当使用 `<browser-*>` 系列宏技能收到类似 `ECONNREFUSED` 或连接失败的系统异常反馈时，绝对禁止尝试使用 `Start-Process`、`curl` 或其他系统命令进行绕过尝试。你必须立即中止当前网页任务，并将具体的异常反馈给用户，要求其检查环境。
6. 回合制交互准则：在进行网页自动化操作时，严禁在同一次回复中连续输出 <browser-scan> 和后续交互动作（如 <browser-action>）。你必须在调用扫描技能后立即结束回复或使用 <continue> 移交控制权，强制等待系统返回真实的元素 ID 列表。严禁凭空捏造网页内容或搜索结果。
